Файловая система Linux

Изучите структуру и организацию файлов в Linux

Иерархия файловой системы

В Linux все файлы и каталоги организованы в единую древовидную структуру, начинающуюся с корневого каталога /.

/ (корень)
bin
Основные исполняемые файлы
etc
Конфигурационные файлы системы
home
Домашние каталоги пользователей
usr
Пользовательские программы и данные
var
Изменяемые файлы (логи, кеши)
tmp
Временные файлы
dev
Файлы устройств
proc
Информация о процессах
lib
Системные библиотеки

Основные каталоги и их назначение

/bin
Содержит основные исполняемые файлы (команды), доступные всем пользователям
/etc
Конфигурационные файлы системы и установленных программ
/home
Содержит домашние каталоги пользователей с их персональными файлами
/usr
Вторичная иерархия для пользовательских данных (программы, библиотеки)
/var
Переменные данные (логи, кеши, базы данных, почта)
/tmp
Временные файлы, которые могут быть удалены при перезагрузке
/dev
Файлы устройств (жесткие диски, принтеры, USB и т.д.)
/proc
Виртуальная файловая система с информацией о процессах и ядре

Права доступа к файлам

В Linux каждому файлу и каталогу назначены права доступа для владельца, группы и всех остальных пользователей.

-rwxr-xr--
Тип: - (обычный файл)
Владелец: rwx (чтение, запись, выполнение)
Группа: r-x (чтение, выполнение)
Остальные: r-- (только чтение)

Команды для работы с правами

chmod
Изменение прав доступа к файлу/каталогу

chmod 755 file - установить права rwxr-xr-x

chmod +x script.sh - добавить право на выполнение

chown
Изменение владельца файла/каталога

chown user file - изменить владельца

chown user:group file - изменить владельца и группу

umask
Установка маски прав по умолчанию

umask 022 - новые файлы будут с правами 755

Ссылки в Linux

В Linux существуют два типа ссылок, которые позволяют обращаться к файлам по разным именам.

Жесткие ссылки (hard links)

  • Создаются командой ln исходный_файл ссылка
  • Ссылаются непосредственно на данные на диске
  • Не могут ссылаться на каталоги
  • Не могут ссылаться на файлы в других файловых системах
  • При удалении оригинала данные остаются доступны через ссылку

Символические ссылки (soft/symbolic links)

  • Создаются командой ln -s исходный_файл ссылка
  • Содержат только путь к оригинальному файлу
  • Могут ссылаться на каталоги
  • Могут ссылаться на файлы в других файловых системах
  • При удалении оригинала ссылка становится нерабочей
