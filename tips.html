Советы для новичков

Полезные рекомендации для тех, кто только начинает работать с Linux

Начало работы

Выбор дистрибутива

Начните с дистрибутивов, дружелюбных к новичкам: Ubuntu, Linux Mint или Fedora. Они имеют хорошую документацию и большое сообщество.

Освойте терминал

Не бойтесь командной строки - это мощный инструмент. Начните с базовых команд (ls, cd, cp, mv) и постепенно расширяйте знания.

Присоединитесь к сообществу

Linux имеет активное сообщество. Ищите помощь на форумах, в чатах и Q&A сайтах, таких как Stack Overflow.

Решения частых проблем

Используйте менеджер пакетов вашего дистрибутива:

  • Ubuntu/Debian: sudo apt install имя_пакета
  • Fedora: sudo dnf install имя_пакета
  • Arch Linux: sudo pacman -S имя_пакета

Если пакета нет в репозиториях, можно скачать .deb/.rpm файл или собрать из исходников.

Попробуйте следующие команды:

  • sudo apt --fix-broken install (для Debian/Ubuntu)
  • sudo dnf autoremove (для Fedora)
  • sudo pacman -Syu (для Arch)

Если проблема сохраняется, поищите решение для конкретного пакета или ошибки.

Используйте команду:

sudo usermod -aG группа имя_пользователя

После этого пользователю нужно выйти и зайти снова, чтобы изменения вступили в силу.

Проверить группы пользователя можно командой groups имя_пользователя.

Используйте команду chmod:

  • chmod 755 файл - владелец: чтение+запись+выполнение, остальные: чтение+выполнение
  • chmod +x файл - добавить право на выполнение
  • chmod -R 755 папка - рекурсивно изменить права для папки и содержимого

Безопасность

Работа с sudo

Не работайте под root постоянно. Используйте sudo для административных задач. Настройте sudoers файл (visudo) для тонкого контроля.

Пароли и SSH

Используйте сложные пароли или SSH-ключи. Отключите вход по паролю в SSH (/etc/ssh/sshd_config). Регулярно обновляйте систему.

Безопасные источники

Устанавливайте ПО только из доверенных репозиториев. Проверяйте PPA перед добавлением. Будьте осторожны с запуском скриптов из интернета.

Шпаргалка по командам

Файлы и каталоги

  • ls - список файлов
  • cp - копирование
  • mv - перемещение/переименование
  • rm - удаление
  • find - поиск файлов

Система

  • top - мониторинг процессов
  • df - свободное место на диске
  • free - использование памяти
  • uname -a - информация о системе
  • reboot - перезагрузка

Сеть

  • ping - проверка соединения
  • ifconfig - сетевые интерфейсы
  • ssh - удаленное подключение
  • wget - скачивание файлов
  • netstat - сетевые соединения

Права доступа

  • chmod - изменить права
  • chown - изменить владельца
  • sudo - выполнить от root
  • passwd - изменить пароль
  • su - сменить пользователя
